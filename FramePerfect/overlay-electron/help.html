FramePerfect HUD

Help

Overview

The overlay shows your inputs as motion + buttons. It can record to .fpkeys and play them back to a virtual gamepad (ViGEm).

Controls

  • Toggle overlayCtrl+Shift+O
  • Start/Stop recordCtrl+Shift+R
  • Save recordingCtrl+Shift+S
  • Import / playCtrl+Shift+I / Ctrl+Shift+P
  • Toggle injectionCtrl+Shift+J
  • Open/Close HelpCtrl+Shift+H • Esc
  • QuitCtrl+Shift+Q

Recording

  1. Press Ctrl+Shift+R. The red dot shows recording.
  2. Perform inputs on keyboard or XInput-compatible controller.
  3. Press Ctrl+Shift+S to save .fpkeys.

Chord window ~40 ms. Buttons render as 12, 23, 123. Diagonals form from U/D + F/B.

Playback

  1. Ctrl+Shift+I and select a .fpkeys file.
  2. Ensure “Inject ON” (toggle Ctrl+Shift+J).
  3. Stop with Ctrl+Shift+P.

Game window must be focused.

Files & Mapping

  • .fpkeys v2: events are { t, type:'down'|'up', key } where keys are W A S D J K M COMMA B V
  • UIJK → 1 2 3 4. Motions: u,d,f,b,uf,ub,df,db.
  • Provider shows xinput+keyboard when a pad was recorded.

Troubleshooting

  • “FpPad exited …” — install ViGEmBus; run app as admin once.
  • No pad events — expose controller as XInput (DS4Windows/Steam Input).
  • HUD clipped — raise overlay height in ensureOverlay().
